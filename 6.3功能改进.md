# 6.3 功能改进记录

## 概览

本次共完成 11 项功能改进，涉及登录状态持久化、面试交互优化、面试流程修复、管理后台增强、错误处理等方向。所有改动均已通过 TypeScript 类型检查和 Vite 生产构建验证。

---

## 1. 用户登录状态持久化（Critical）

**文件**: `frontend/src/store/index.ts`

**改动内容**:
- 引入 `zustand/middleware` 的 `persist` 中间件
- 将 `token`、`user`、`theme` 三个字段持久化到 `localStorage`（key: `auth-storage`）
- 刷新页面后登录状态不再丢失，无需重新登录

**使用方式**: 无需额外操作，自动生效。用户登录后刷新页面，状态自动恢复。

---

## 2. 用户输入区域交互优化

**文件**: `frontend/src/pages/Interview.tsx`

**改动内容**:
- 将自定义拖拽伸缩句柄从右下角迁移至右上角，参考 Gemini 风格设计
- 拖拽句柄由两条灰色横线组成，hover 时变为蓝色，提供清晰的拖拽提示
- 支持鼠标拖拽上下调整 textarea 高度，范围 52px ~ 400px
- 拖拽过程中禁用页面文本选择，防止误选
- 保持 textarea 自动高度调整功能（输入内容时自动扩展至 200px 上限）

**技术实现**:
- 通过 `onMouseDown` / `mousemove` / `mouseup` 事件实现自定义拖拽
- 使用 `document.body.style.userSelect` 和 `cursor` 优化拖拽体验
- 使用 `isResizingRef`、`startDragYRef`、`startHeightRef` 三个 ref 跟踪拖拽状态

## 3. 面试流程逻辑修复

**文件**:
- `backend/app/services/interview_config.yaml`
- `backend/app/services/interview_config.py`
- `backend/app/services/prompt_builder.py`

### 3.1 面试轮次恢复
**问题**: 测试脚本曾将 YAML 配置覆盖为 12 轮 / 3 阶段，导致面试提前结束。
**修复**:
- 将 `interview_config.yaml` 恢复为 15 轮 / 5 阶段配置
- 同步更新 `interview_config.py` 中的 `_DEFAULT_CONFIG`
- 5 个阶段依次考察：逻辑表达 → 专业知识广度 → 技术深度 → 项目实践能力 → 应变与解决问题能力 → 沟通协作素养 → 岗位匹配度

### 3.2 系统提示词增强
**问题**: LLM 有时会过早结束面试，在第 7-8 轮就输出结束语。
**修复**: 在 `prompt_builder.py` 的系统提示词模板中添加了强约束规则：
- 添加 🔴 标记的防提前结束规则，明确禁止 LLM 在非最后一轮生成结束语
- 引入 `remaining_turns` 模板变量，让 LLM 清楚地知道剩余轮数
- 在每一阶段的提示词中强调当前阶段还在继续，不应收尾

## 4. 面试中"停止生成"按钮

**文件**: `frontend/src/pages/Interview.tsx`

**改动内容**:
- 当 AI 正在流式生成回复时（`isStreaming === true`），发送按钮替换为红色停止按钮（StopCircle 图标）
- 点击停止按钮后：
  1. 通过 `AbortController.abort()` 中止当前 SSE 请求
  2. 将已生成的部分内容保存为一条 assistant 消息
  3. 清空流式状态，恢复输入框可用

**使用方式**: AI 生成回复过程中，右下角按钮自动变为红色停止图标，点击即可。

## 6. 管理员表格排序/筛选/批量操作

**文件**: `frontend/src/pages/Admin.tsx`

### 6.1 列排序
- 新增 `SortButton` 组件和 `useSortState` Hook，支持三态切换（升序 → 降序 → 取消）
- 已添加排序的表格：
  - **用户管理**: 用户名、面试次数、均分、最近面试时间

- 排序为客户端排序，在当前页面数据上执行

### 6.2 批量操作（题库管理）
- 表格新增复选框列，支持单选和全选
- 选中题目后工具栏出现"删除选中 (N)"按钮
- 点击执行批量删除，逐个调用删除 API，完成后显示成功/失败数量

## 技术细节

| 项目 | 说明 |
|------|------|
| Zustand persist | key: `auth-storage`，仅持久化 `token`、`user`、`theme` |
| 拖拽伸缩句柄 | 自定义 mousedown/move/up 事件，参考 Gemini 右上角设计 |
| 面试轮次 | YAML 配置 15 轮 5 阶段，系统提示词含防提前结束约束 |
| SSE 中断机制 | 通过 `AbortController` 中止 fetch 请求，保存部分内容 |
| 排序逻辑 | 客户端排序，支持中文字符串 `localeCompare("zh-CN")` |
| Error Boundary | React Class Component，捕获渲染错误，不捕获事件处理 |
| 骨架屏 | 使用 Tailwind `animate-pulse` + `bg-slate-200` 实现 |
