# AceInterviewer 项目特色

AceInterviewer 不是简单把简历丢给大模型做问答，而是把“简历解析、岗位要求、结构化面试流程、题库检索、实时进度、证据化评估报告”串成一个完整的模拟面试产品。

## 核心特色

### 1. 简历 + JD 双输入的岗位定制面试

平台支持上传 PDF 简历，并额外填写目标岗位名称和岗位要求描述。系统会同时参考：

- 简历中的技术栈关键词
- 简历原文中的项目经历
- 用户粘贴的 JD 职责、必备技能和加分项

这样生成的问题不只是围绕简历泛泛追问，而是会关注“候选人经历和目标岗位是否匹配”。

### 2. 用户可选择面试风格

面试开始前，用户可以选择本次面试风格：

- 基础技术面：偏常规技术考察，关注基础、项目和工程实践
- 压力面试：偏高强度追问，关注边界条件、故障排查和抗压表现
- 轻松聊天：偏同行交流，关注项目复盘、表达和职业发展

相比直接和大模型聊天，平台把“面试官风格”产品化为明确选项，用户可以按真实面试场景训练。

### 3. 面试中实时显示考察进度

面试页会展示结构化考察进度，包括：

- 当前进行到第几轮
- 当前问题主要考察的能力维度
- 已覆盖的技术点或能力点
- 待考察的技术点或能力点

这让用户清楚知道自己正在被考察什么，也能看到面试不是随机聊天，而是按能力模型逐步推进。

### 4. RAG 题库增强提问

平台内置题库和知识库能力，管理员可以维护题目、知识片段和文档。面试时系统会根据简历关键词检索相关题目，作为大模型提问参考。

这使面试问题更稳定、更贴近平台定义的考察标准，而不是完全依赖大模型临场发挥。

### 5. 六维能力模型评分

面试结束后，系统会生成六维雷达评分：

- 技术深度
- 逻辑表达
- 专业知识广度
- 应变与解决问题能力
- 沟通与协作素养
- 项目实践能力

评分不是只给一句“表现不错”，而是把面试表现拆解成可比较、可复盘的能力维度。

### 6. JD 岗位匹配报告

如果用户填写了岗位要求，报告中会额外生成：

- 岗位匹配度
- JD 匹配亮点
- JD 差距分析
- 岗位补强建议

这让报告从“面试表现评价”进一步升级为“针对目标岗位的求职准备建议”。

### 7. 证据化反馈报告

报告会引用候选人回答中的具体片段，指出对应结论和改进方向。例如：

- 哪一轮回答体现了某个技术能力
- 哪一轮回答没有解释清楚项目结果
- 哪些 JD 要求没有被回答证明

这比普通大模型总结更可信，因为每条评价都有对话证据支撑。

### 8. 历史面试记录与报告沉淀

平台会保存历史面试会话和评估报告，用户可以回看不同时间、不同面试类型下的表现。

这让平台具备持续训练价值，而不是一次性的聊天记录。

### 9. 管理后台与可运营能力

项目包含管理员后台，支持：

- 题库管理
- 知识库文档管理
- Prompt 版本管理
- 数据统计仪表盘
- 操作日志审计

这说明平台不仅是一个 Demo 聊天页面，而是具备内容维护、质量控制和运营扩展能力的完整系统。

## 与直接使用 DeepSeek 官网上传简历的区别

| 对比项 | 直接使用 DeepSeek 官网 | AceInterviewer |
| --- | --- | --- |
| 使用方式 | 用户自己组织提示词和追问方向 | 平台提供完整面试流程 |
| 简历理解 | 依赖一次上传后的自由问答 | 自动解析简历关键词和原文 |
| JD 适配 | 需要用户手动提示 | 内置岗位名称和 JD 输入，面试与报告都会参考 |
| 面试风格 | 需要用户自己描述 | 用户可直接选择基础技术面、压力面试、轻松聊天 |
| 面试过程 | 普通聊天流 | 分阶段面试 + 实时考察进度 |
| 题目来源 | 完全依赖模型生成 | 可结合平台题库和知识库 RAG |
| 评估方式 | 通常是总结式反馈 | 六维雷达评分 + JD 匹配报告 |
| 反馈可信度 | 容易泛泛而谈 | 证据化反馈引用具体回答片段 |
| 数据沉淀 | 聊天记录零散 | 历史面试和报告可持续查看 |
| 管理能力 | 无平台管理后台 | 支持题库、知识库、Prompt、审计和统计 |

## 项目定位

AceInterviewer 的核心价值是把大模型能力封装成一个可重复、可配置、可评估的 AI 面试训练系统。它不是替用户“问大模型一个问题”，而是帮助用户完成一次接近真实招聘流程的模拟面试，并输出可复盘、可改进的结构化结果。
